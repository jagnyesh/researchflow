# Root Cause Analysis: Count Mismatch for Diabetes Demographics Request

**Date:** November 10, 2025
**Request:** "I need demographics(family name, given name, date of birth) of all female patients with diabetes."
**Request ID:** REQ-20251110-B68EC618
**Issue:** Count mismatch between manual SQL query (18 patients) and expected ResearchFlow results

---

## Executive Summary

**Root Cause:** Case-sensitive LIKE matching in manual SQL query
**Impact:** Manual query misses 1 valid patient (326492 - Schneider199)
**Correct Answer:** **19 patients** (not 18)
**ResearchFlow Status:** ✅ **WORKING CORRECTLY** (uses case-insensitive LOWER())

---

## Diagnostic Results

### Finding #1: Case Sensitivity Issue

**Manual Query (from screenshot):**
```sql
SELECT DISTINCT pd.name_family, pd.name_given, pd.dob
FROM sqlonfhir.patient_demographics pd
INNER JOIN sqlonfhir.condition_simple cs ON pd.patient_id = cs.patient_id
WHERE pd.gender = 'female' AND cs.icd10_display LIKE '%diabetes%';
-- Returns: 18 rows
```

**Problem:** Case-sensitive `LIKE '%diabetes%'` does NOT match "**D**iabetes" (capital D)

**ResearchFlow Query (generated by sql_generator.py):**
```sql
SELECT COUNT(DISTINCT p.patient_id) as patient_count
FROM sqlonfhir.patient_demographics p
WHERE EXISTS (
    SELECT 1 FROM sqlonfhir.condition_simple c
    WHERE c.patient_id = p.patient_id
    AND LOWER(c.code_text) LIKE LOWER('%diabetes%')  -- ✅ Case-insensitive
)
AND p.gender = 'female';
-- Returns: 19 patients (CORRECT)
```

### Finding #2: Missing Patient Identified

**Patient 326492 (Schneider199):**
- Has condition: "**D**iabetes mellitus type 2 (disorder)" (capital D)
- Does NOT have any other diabetes conditions with lowercase 'd'
- ❌ Missed by manual query (case-sensitive)
- ✅ Found by ResearchFlow (case-insensitive)

**Other patients with capital-D also have lowercase conditions:**
- 149474 (Deckow585): Capital-D + "Prediabetes" → Counted in both queries
- 307901 (Cremin516): Capital-D + "Prediabetes" + complications → Counted in both queries
- 380914 (Daugherty69): Capital-D + "Prediabetes" → Counted in both queries

### Finding #3: Column Usage

**Table:** `sqlonfhir.condition_simple`
**Available Columns:**
- `icd10_code` (indexed) - e.g., "44054006"
- `icd10_display` - e.g., "Diabetes mellitus type 2 (disorder)"

**Current Usage:**
- Manual query: Uses `icd10_display`

**Analysis:** Both columns contain identical data (verified in diagnostic 7)

### Finding #4: SELECT DISTINCT Behavior

**Test:** Does SELECT DISTINCT on multiple columns affect count?

```sql
-- COUNT DISTINCT patient_id only
SELECT COUNT(DISTINCT pd.patient_id) → 18 patients

-- COUNT with DISTINCT on all columns
SELECT COUNT(*) FROM (
    SELECT DISTINCT pd.patient_id, pd.name_family, pd.name_given, pd.dob ...
) → 18 patients
```

**Result:** ✅ SELECT DISTINCT is NOT causing count mismatch (both return 18)

### Finding #5: NULL Field Analysis

**All 19 female diabetes patients:**

- ❌ Have NULL `given_name` (19 NULL values, 100%)

**Impact:** Separate data quality issue, does NOT affect count

**Lines 301-326:** Condition clause builder

```python
def _build_condition_clause(self, condition_term: str, include: bool) -> Tuple[str, Dict[str, Any]]:
    operator = "EXISTS" if include else "NOT EXISTS"
    param_name = self._get_param_name("condition")
    params = {param_name: f"%{condition_term}%"}

    condition_table = self._build_table_name(self.condition_table)
    condition_col = self.condition_code_column  # ← "code_text"

    sql = f"""{operator} (
        SELECT 1 FROM {condition_table} c
        WHERE c.patient_id = p.{patient_id_col}
        AND LOWER(c.{condition_col}) LIKE LOWER(:{param_name})  # ← ✅ Uses LOWER()
    )"""
    return sql, params
```

**Security:** ✅ Uses parameterized queries (line 323)

## Conclusions

### 1. ResearchFlow is CORRECT

**Evidence:**
- Uses case-insensitive matching with `LOWER()`
- Returns **19 patients** including Schneider199
- Follows SQL best practices for text matching

### 2. Manual Query is INCORRECT

**Problem:**
```sql
WHERE cs.icd10_display LIKE '%diabetes%'  -- ❌ Case-sensitive
```

**Fix:**
```sql
WHERE LOWER(cs.icd10_display) LIKE '%diabetes%'  -- ✅ Case-insensitive
```

### 3. Column Choice is Optional

**Both columns work correctly:**
- `code_text`: Generic FHIR field (current ResearchFlow usage)
- `icd10_display`: ICD-10 specific field (manual query usage)

**Recommendation:** Prefer `icd10_display` for semantic clarity, but NOT required
